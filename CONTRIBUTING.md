# Contributing to open-defender-icap

Thank you for your interest in contributing to open-defender-icap.

## Before you contribute

Please review:

- `README.md`
- `SECURITY.md`
- `TRADEMARKS.md`

## Ways to contribute

- report bugs
- improve documentation
- propose features
- submit fixes and enhancements
- help with tests and examples

## Reporting bugs

When opening an issue, please include:

- environment details
- exact steps to reproduce
- expected behavior
- actual behavior
- logs or traces, if available

## Pull requests

Please keep pull requests focused and well-described.

Include:

- what changed
- why the change is needed
- how it was tested
- any compatibility or migration impact

## Code quality

By submitting a contribution, you confirm that:

- you have the right to submit the contribution
- the contribution does not knowingly violate third-party rights
- you are submitting it under the same project license terms

## Security issues

Do not file public issues for suspected vulnerabilities.
Report them according to `SECURITY.md`.

## Project governance

open-defender-icap is maintained by TREX FZE.
TREX FZE retains final decision-making authority over roadmap, releases, and acceptance of contributions.
